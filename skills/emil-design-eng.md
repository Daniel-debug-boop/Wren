# Emil Kowalski Design Engineering Philosophy

The following is extracted from Emil Kowalski's skills repository (github.com/emilkowalski/skills). These principles inform every design decision made here.

## Taste Is The Edge

When AI and no-code tools make building trivial, **taste becomes the differentiator**. The app that feels right wins over the one that simply works.

## The Unseen Details

Great software is shaped by what users never consciously notice:

- A 2px border-radius that matches every other corner
- A transition that never takes longer than 300ms
- The one grey that sits perfectly between text and background
- Typography that breathes — tracking, weight, contrast
- Icons that share a single stroke weight

## Beauty Is Leverage

When a product looks crafted, users assume the code is solid, the data is safe, the company is credible. Every pixel signals trust.

## Core Principles

1. **Subtraction over addition.** Remove until the last meaningful element. If you can delete an animation, delete it.
2. **Consistency compounds.** A design system only works if every component obeys the same rhythm: spacing tokens, type scale, border radius, shadow depth.
3. **Motion communicates.** Every animation must answer "why does this animate?" — spatial consistency, state indication, feedback, or preventing a jarring change.
4. **Constraints breed craft.** Limit your palette: 2 typefaces max, 3 colors, 1 shadow scale. Master those before adding more.
5. **The 100px rule.** The most important 100px on any screen determines how the whole product feels. Get that hero, that header, that first interaction perfect — it anchors everything.

## Practical Application

- **Audit existing UI first.** Find the inconsistencies: mixed border radii, inconsistent padding, misaligned baselines.
- **Fix before adding.** A polished existing screen is worth more than a new half-baked one.
- **Design tokens are non-negotiable.** Every color, spacing value, font size, and shadow should reference a token. Hardcoded values are technical debt.
- **White space is structure.** Don't fill space — let it create hierarchy and focus.
